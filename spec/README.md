# 实验室知识库 Spec

版本: v1.0

本目录用于沉淀实验室知识库项目的规格、行为规范、验收标准、实现路线和测试用例设计。所有具体实现工作都必须先在这里完成 Spec 讨论和确认。

## 文档索引

- `behavior-rules.md`: 项目协作与工程行为规范。
- `mvp-spec.md`: 第一阶段实验室知识库 MVP 实现规格。
- `implementation-plan.md`: 整体实施路线图与阶段验收标准。
- `implementation-status.md`: 当前实施状态、验证结果和剩余风险。
- `knowledge-base-build-spec.md`: 当前实验室 SOP 知识库构建结果规格。
- `test-accumulation.md`: 测试用例积累规范与评测资产格式。

## 当前阶段状态

当前处于第一阶段核心搭建已完成、待继续迭代状态。

第一阶段目标是在 `lab-rag-qa/` 项目中，基于两份 PDF 构建一个可测试、可检索、可问答、可溯源的实验室知识库。`RAG-cy/` 作为参考项目，借鉴 PDF 解析、父文档检索、重排和结构化回答思路，但不直接照搬竞赛代码。

## 已确认决策

- 交付形态: `CLI + HTTP API` 为主，UI 作为跟随演示层。
- 文档来源: 第一阶段只使用两份 PDF。
- 解析策略: 本地解析优先，MinerU 作为可选增强。
- 检索路线: BM25/关键词检索 + 本地规则重排。
- 引用粒度: 章节 + 页码 + chunk。
- 生成策略: 本地抽取式兜底 + 可选 Qwen/LLM 生成。
- 默认语言: 中文优先，英文术语和实验参数保留原文。
- 评测门槛: 第一阶段建立 10 问冒烟黄金集。

## 工作约定

1. 先进行 Spec 探讨，再进入具体实现。
2. 所有工作产出都必须同步积累测试用例。
3. 后续代码实现必须优先满足 `mvp-spec.md` 和 `implementation-plan.md`。
4. 每次实现交付都必须说明测试覆盖、测试结果和未覆盖风险。
